# Agentic Personal Knowledge Base

[![README zh-CN](https://img.shields.io/badge/README-zh--CN-blue)](README_zh.md)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

An agentic Obsidian vault architecture for building a self-maintaining personal
knowledge base.

This project is more than a folder template. It defines a rules-first operating
system for AI agents: new files enter through an inbox, routing decisions are
made from explicit category criteria, notes are normalized through templates,
multi-format sources are converted before cleanup, indexes are generated by
Dataview, and a validator checks the result.

## Why This Design

Most AI-assisted note workflows rely on one-off prompts. That works for a few
files, but it becomes fragile as the vault grows. This skeleton turns the vault
itself into the control plane.

- **Rule-driven routing:** `RULES.md` and category `README.md` files act as a
  routing table. Agents classify inbox files by reading explicit criteria rather
  than guessing from memory.
- **Safe fallback path:** uncertain files stay in `00-Inbox/` with
  `status: needs-review`, so ambiguity does not silently corrupt the knowledge
  base.
- **Template-based normalization:** `.templates/BASE_TEMPLATE.md` and category
  `.TEMPLATE.md` files turn raw content into consistent Markdown with frontmatter,
  status, tags, source metadata, remarks, and related links.
- **Multi-format ingestion:** PDFs, Office files, HTML, CSV, JSON, EPUB, images,
  audio, and archives can be converted with MarkItDown first, then cleaned and
  organized by the agent.
- **Automatic layout and indexing:** agents maintain note metadata; Dataview
  renders the index. There is no hand-maintained contents table to drift out of
  sync.
- **Deterministic maintenance:** `.kb-maintenance/validate_vault.py` checks root
  files, category structure, templates, frontmatter, tags, asset links, file
  names, and progress-log shape.
- **Personal context layer:** `SOULS.md` is designed as a compact user-profile
  layer for private vaults. It gives agents long-term context about the owner's
  preferences and active interests without requiring a full vault reread on every
  task.

## Workflow

```text
00-Inbox/
  -> Detect
  -> Read
  -> Classify
  -> Convert
  -> Format
  -> Move
  -> Dataview Index
  -> Validate
```

The interesting part is that each step has a fixed responsibility:

| Step | Responsibility |
| --- | --- |
| Detect | Find new inbox files and skip irrelevant system files |
| Read | Inspect complete source content before making decisions |
| Classify | Route files by category criteria |
| Convert | Use MarkItDown for non-Markdown source formats |
| Format | Apply base and category templates |
| Move | Place notes and assets in stable vault locations |
| Index | Let Dataview query frontmatter instead of editing tables |
| Validate | Run deterministic structure and metadata checks |

## Repository Layout

```text
.
├── README.md              # Public GitHub overview
├── README_zh.md           # Chinese overview
├── AGENTS.md              # Canonical agent entrypoint
├── CLAUDE.md              # Symlink to AGENTS.md
├── GEMINI.md              # Symlink to AGENTS.md
├── RULES.md               # Routing, formatting, and maintenance rules
├── CONTENTS.md            # Dataview-based automatic index
├── SOULS.md               # User-profile placeholder for private vaults
├── .templates/            # Note templates
│   ├── README.md
│   └── BASE_TEMPLATE.md
├── .kb-maintenance/       # Maintenance scripts
│   ├── README.md
│   └── validate_vault.py
├── 01-Example/            # Synthetic example category
│   ├── README.md
│   ├── .TEMPLATE.md
│   └── SampleNote.md
├── assets/                # Fixed attachment store
└── 00-Inbox/              # Intake folder for unprocessed files
```

`README.md` is the project introduction. Agents should start from `AGENTS.md`;
`RULES.md` is the detailed operational source of truth.

## Quick Start

Clone the repository:

```bash
git clone <repository-url>
cd Agentic-Personal-Knowledge-Base
```

Validate the skeleton:

```bash
python3 .kb-maintenance/validate_vault.py --strict-tags --strict-names
```

Expected result:

```text
VALIDATION_STATUS=PASS
```

Optional source-file conversion support:

```bash
pip install "markitdown[all]"
```

Optional Obsidian setup:

- Open the folder as an Obsidian vault.
- Install the Dataview community plugin if you want `CONTENTS.md` to render live tables.

## Creating Categories

The included `01-Example/` directory demonstrates the category contract. A normal
category looks like this:

```text
NN-CategoryName/
├── README.md
└── .TEMPLATE.md
```

The category `README.md` should include an `Agent Classification Criteria`
section. That section is what turns categories into an automatic routing layer.
